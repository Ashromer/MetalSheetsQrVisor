# Proyecto: Visor 3D + Catálogo Web — Metalperfil

## Qué es esto
Web alojada en GitHub Pages (repo `MetalSheetsQrVisor`, usuario `Ashromer`) para una
empresa de revestimientos metálicos. Tiene dos partes:

1. **`index.html`** — Visor 3D interactivo. Muestra perfiles metálicos en 3D con
   `<model-viewer>`, permite elegir familia, espesor y color, y enlaza al catálogo.
2. **`catalogo.html`** — Catálogo web con todas las familias de producto (descripción,
   dimensiones, materiales, galería de imágenes). **Data-driven**: el contenido se edita
   en el array `FAMILIES` del `<script>` al final del archivo.

URL publicada: https://ashromer.github.io/MetalSheetsQrVisor/

## Stack
- HTML/CSS/JS plano, sin framework ni build. Todo en archivos sueltos.
- `<model-viewer>` vía CDN jsDelivr (`@google/model-viewer`).
- Tipografías Google Fonts: Playfair Display (serif) + Inter (sans).
- Color corporativo granate: **`rgb(150, 24, 30)`** (oscurecido a petición del cliente;
  antes era `#B61F26`). Definido como `--brand-red-granate` en ambos HTML.

## ⚠️ No tocar sin cuidado (en index.html)
El visor costó hacerlo funcionar. Claves que NO hay que cambiar:
- El CDN debe ser jsDelivr (`@google/model-viewer`), no el de googleapis.
- El evento `load` del model-viewer SOLO debe ocultar el spinner (+ aplicar color). NO añadir
  manipulación de cámara (`cameraOrbit`, `jumpCameraToGoal`, `getBoundingBoxCenter`)
  dentro del `load`: provoca un cuelgue ("rAF timed out") y el modelo no se ve.
- El ángulo de cámara se fija con el atributo `camera-orbit="35deg 75deg auto"` en el HTML.
- `.viewer-container` necesita una `height` fija (vh), no `flex`.
- `disable-pan` debe quedarse.

## Cómo completar el catálogo (catalogo.html)
Todo el contenido vive en el array `FAMILIES` y los nodos del `<script>` final:
- Para rellenar una familia: poner `placeholder: false` y añadir `desc`, `materials`,
  `dims: [{tipo, ancho, altura}]`, `images: ['img/...']` y opcional `glb: 'Archivo.glb'`.
- Las familias con `placeholder: true` se renderizan con aviso rojo "TEXTO PROVISIONAL" y
  huecos `img-slot`.
- Las secciones Origen / Trajes a medida / Materialidad / BIM tienen su texto provisional
  marcado con `<span class="placeholder-note">` directamente en el HTML.
- Anclas (para enlazar desde el visor): `#pyramid`, `#symmetric`, `#asymmetric`,
  `#escaler`, `#kaotico`, `#origami`, `#onda`, `#kubo`, `#nordik`, `#alterno`, `#skala`.

## Inconsistencias de nomenclatura (cuidado al enlazar)
- Catálogo usa **Asymmetric** y **Escaler**; el visor/los .glb usan **Asimmetric** y
  **Eskaler**. El botón del visor ya mapea cada familia a su ancla correcta del catálogo
  (`catalog[familia].anchor` en index.html → `catalogo.html#asymmetric`, etc.).

## Reproducir la extracción
Requiere Python 3 + `pip install PyMuPDF Pillow`. Desde la raíz del repo:
`python tools/extract_pdf.py` (imágenes + texto) y `python tools/trim_logo.py` (logo).
